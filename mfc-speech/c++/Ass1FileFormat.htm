Speech File Content

There are 3x3 files for the training of the automatic speech recognition system:

3 speakers: s014, s017 and s029 speak each of the digits "one", "two" and "three". Accordingly, the 9 training files are named s014d001.mfc ... s029d003.mfc.

There is one test file which contains an unknown digit between "one" and "three", spoken by speaker s031. This file is named s031d00x.mfc and the utterance in this  file is to be "recognised" by the automatic speech recognition system.

Speech File Format

The files contain sets of mel-frequency cepstral coefficients ( MFCCs) computed using the HMM Toolkit (HTK). Each file contains 1 line with 4 integer values, followed by 1 line for each frame with 11 float values.

The first line of the file contains the 4 values:
1. the number of frames in the file
2.  the time period represented by 1 frame in units of 100ns (this is 100,000, being a period of 100,000 x 100ns = 10,000,000 ns = 0.01s)
3.  the number of bytes for each frame (this is 44, representing 11 float numbers)
4. a code for  the kind of parameters following (this is octal 14106, standing for mel-frequency cepstral coefficients (MFCCs)  and log energy)

For the current application, only the number of frames in the file is of importance for the subsequent reading of the frame vectors.

The following lines each contain a frame vector, comprising
ten MFCCs c(1) to c(10) and the log energy.
